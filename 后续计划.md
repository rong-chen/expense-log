## 一、标签系统 (Tag System)

### 1.1 功能描述
在现有固定分类（餐饮/交通/购物等）之外，支持用户自定义标签，实现更灵活的账单归类和检索。

### 1.2 应用场景
- `#出差` — 出差期间所有消费打上此标签，月底一键筛选报销
- `#AA` — 标记需要和朋友分账的消费
- `#报销` — 公司报销相关的消费追踪
- `#旅游-日本` — 按旅行目的地归档消费

### 1.3 技术方案

#### 后端
- **新增表 `tags`**
  ```sql
  CREATE TABLE tags (
    id UUID PRIMARY KEY,
    user_id UUID NOT NULL REFERENCES users(id),
    name VARCHAR(50) NOT NULL,
    color VARCHAR(20) DEFAULT '#3498db',  -- 标签颜色
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(user_id, name)
  );
  ```
- **新增关联表 `bill_tags`**（多对多）
  ```sql
  CREATE TABLE bill_tags (
    bill_id UUID NOT NULL REFERENCES bills(id) ON DELETE CASCADE,
    tag_id UUID NOT NULL REFERENCES tags(id) ON DELETE CASCADE,
    PRIMARY KEY (bill_id, tag_id)
  );
  ```
- **API 端点**
  - `GET /api/v1/tags` — 获取当前用户所有标签
  - `POST /api/v1/tags` — 创建标签 `{ name, color }`
  - `DELETE /api/v1/tags/:id` — 删除标签
  - `POST /api/v1/bill/:id/tags` — 给账单添加/移除标签 `{ tag_ids: [] }`
  - `GET /api/v1/bills?tag=xxx` — 按标签筛选账单

#### 前端
- **标签管理页**：在「我的」中增加「标签管理」入口，支持创建/删除/选色
- **记账时选标签**：在 BillAdd / BillEdit 页面增加标签多选区域
- **列表筛选**：Bills 页面增加标签筛选 chip
- **AI 识别扩展**：prompt 中可引导 AI 根据内容自动推荐匹配标签

### 1.4 优先级
⭐⭐⭐ 中等优先级 — 实现复杂度适中，用户体验提升明显

## 二、共享账本 (Shared Ledger)

### 2.1 功能描述
支持多用户共同管理同一份账本，适用于家庭、情侣、室友等场景。每个用户可以同时拥有「个人账本」和多个「共享账本」。

### 2.2 应用场景
- **家庭账本** — 夫妻共同记录家庭开支，各自通过手机录入
- **室友合租** — 共同记录水电房租等公共开支，月底自动计算分摊
- **项目报销** — 团队成员共同记录项目开支

### 2.3 技术方案

#### 后端
- **新增表 `ledgers`**（账本）
  ```sql
  CREATE TABLE ledgers (
    id UUID PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    owner_id UUID NOT NULL REFERENCES users(id),  -- 账本创建者
    type VARCHAR(20) DEFAULT 'shared',  -- 'personal' | 'shared'
    created_at TIMESTAMP DEFAULT NOW()
  );
  ```
- **新增表 `ledger_members`**（账本成员）
  ```sql
  CREATE TABLE ledger_members (
    ledger_id UUID NOT NULL REFERENCES ledgers(id) ON DELETE CASCADE,
    user_id UUID NOT NULL REFERENCES users(id),
    role VARCHAR(20) DEFAULT 'member',  -- 'owner' | 'admin' | 'member'
    joined_at TIMESTAMP DEFAULT NOW(),
    PRIMARY KEY (ledger_id, user_id)
  );
  ```
- **修改 `bills` 表**：新增 `ledger_id` 字段
  ```sql
  ALTER TABLE bills ADD COLUMN ledger_id UUID REFERENCES ledgers(id);
  -- NULL 表示个人默认账本
  ```
- **API 端点**
  - `POST /api/v1/ledgers` — 创建共享账本
  - `GET /api/v1/ledgers` — 获取我参与的所有账本
  - `POST /api/v1/ledgers/:id/invite` — 邀请成员（通过邀请码）
  - `DELETE /api/v1/ledgers/:id/members/:uid` — 移除成员
  - `GET /api/v1/ledgers/:id/bills` — 获取指定账本的账单
  - `GET /api/v1/ledgers/:id/stats` — 共享账本的统计数据

#### 前端
- **账本切换器**：首页顶部增加账本切换下拉（个人 / 家庭 / ...）
- **账本管理页**：创建共享账本、邀请成员、管理权限
- **记账归属**：记账时可选择归入哪个账本
- **成员贡献视图**：共享账本中展示各成员的支出占比
- **分摊计算**（可选进阶功能）：自动计算每人应分摊金额

### 2.4 关键设计决策
- **数据隔离**：共享账本的账单对所有成员可见，但只有录入者和管理员可编辑/删除
- **权限模型**：owner（全权管理）> admin（可管理成员）> member（仅记账和查看）
- **默认账本**：每个用户注册时自动创建一个「个人账本」，`ledger_id = NULL` 的历史账单归入个人账本
- **退出机制**：成员退出共享账本后，其录入的账单保留在共享账本中

### 2.5 优先级
⭐⭐ 较高复杂度 — 涉及权限体系、数据隔离、多端同步，建议在标签系统完成后实施

## 实施顺序建议

```
Phase 1: 标签系统
  └── 后端模型 + API → 前端标签管理 → 记账页集成 → 列表筛选

Phase 2: 共享账本
  └── 账本模型设计 → 成员/权限系统 → 前端账本切换 → 共享统计视图
```
